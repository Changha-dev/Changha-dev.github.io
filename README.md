# Notion to Hugo sync

This repository now includes a Notion sync script that reads posts from a Notion data source and writes Hugo markdown files into `content/posts`.

## What it does

- Queries a Notion data source and reads page properties plus block content
- Generates Hugo front matter and markdown files in `content/posts/<slug>.md`
- Downloads page cover images and image blocks into `content/posts/<slug>-img/`
- Protects manually written posts by only overwriting files that were previously generated by this sync

## Setup

1. Create a Notion internal integration and copy its API token.
2. Share your blog data source with that integration in Notion.
3. Copy `.env.example` to `.env`.
4. Fill in `NOTION_API_TOKEN` and either:
   - `NOTION_DATA_SOURCE_ID`
   - or `NOTION_DATABASE_ID` if the database contains exactly one data source
5. Run:

```bash
npm run notion:sync
```

## Recommended Notion properties

Only the title property is strictly required. These property names are configurable in `.env`.

- `Name` (title)
- `Slug` (rich text)
- `Date` (date)
- `Description` (rich text)
- `Tags` (multi-select)
- `Categories` (multi-select)
- `Topics` (multi-select)
- `Status` (status or select)
- `Draft` (checkbox, optional)

## Publishing behavior

- When `NOTION_SYNC_PUBLISHED_ONLY=true`, only pages whose `Status` equals `Published` are synced.
- If `NOTION_DRAFT_PROPERTY` exists and is checked, the generated post uses `draft: true`.
- If a page has no `Slug`, the script generates one from the title.

## Supported content blocks

The sync handles the common Notion blocks used in blog posts:

- Paragraphs
- Headings
- Bulleted and numbered lists
- To-do items
- Quotes and callouts
- Code blocks
- Dividers
- Images and files
- Bookmarks and link previews
- Toggles
- Tables

Unsupported block types are left as HTML comments in the generated markdown so the sync does not silently drop content.

## Notes

- Generated posts include a `notion_page_id` front matter field so the sync can safely update them later.
- If your Notion database has multiple data sources, set `NOTION_DATA_SOURCE_ID` explicitly.
- Notion file URLs are temporary; the sync downloads them into the repo so your Hugo site can serve them directly.
